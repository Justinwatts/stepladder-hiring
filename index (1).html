about

Stepladder builds custom AI software for small businesses that need it but can't navigate it alone. We find where the leverage is and put working software into their operation.

First live build: an agent for an HVAC business that turns inbound email into jobs in their field-service system. More client work is right behind it. We're a focused team now, building toward something bigger, which is why the ground floor is worth getting in on.

the role

The founder defines what needs building and owns the client relationships. The how is yours: architecture, tooling, trade-offs, and the technical decisions that come with them. You make them, not just implement them.

As the first engineer, you'll set the technical foundation and grow into broader technical and strategic work as we do. Critical part of the team from day one.

how we work

Goals, not specs

We define what good looks like. You decide how to get there.

AI agents are a primary tool

We build with Claude Code. A core part of the job is reviewing AI-generated code for the bugs that don't surface in testing: race conditions, unhandled errors, security holes, silent data problems.

Disciplined, not heavy

Linear for tickets, clean version control, review before ship. Process that helps, not process for its own sake.

Pragmatic shipping

Ship to client acceptance and move on. Knowing which corners are safe to cut is a core skill.

You shape how we build

The dev system is young. Scaffolds, templates, automation, standards: you'll have direct influence over all of it.

what you'll do
  • Own the technical approach end-to-end, from requirements to running software.
  • Build with AI tooling, then review and validate the output before it ships.
  • Build and maintain third-party API integrations. The core of most projects.
  • Own deployment, secrets, observability, and reliability for what you build.
  • Run the technical testing touchpoints with clients. The founder leads the relationship.
  • Improve how we build: scaffolds, templates, automation, CI.
requirements
  • Have 3–5 years professional software engineering experience.
  • Can find what's wrong in code you didn't write. Strong on review and debugging.
  • Can take a loosely-specified goal and design the solution yourself.
  • Are solid with REST API integrations: OAuth, webhooks, polling, rate limits, idempotency, messy real-world failures.
  • Are comfortable across Python and JavaScript / TypeScript.
  • Have deployed and run production apps on a modern PaaS.
  • Write clear English and communicate well with non-technical people.
  • Are comfortable in a ticket-driven process, and want to improve it.

Bonus: hands-on with AI coding agents (Claude Code, Cursor, or similar) · you've handled production incidents and can talk root-cause · self-directed in a remote, async, cross-timezone team · you have a point of view on how things should be built.
